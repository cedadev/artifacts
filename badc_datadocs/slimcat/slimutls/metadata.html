  • Model type

    Three-dimensional (3-D) chemistry transport model (CTM).

  • Model description

    Please see http://www.lec.leeds.ac.uk/~martyn/slimcat.html.

  • Model run

    Start time: October 1991.
    Forcing: UK Met Office analyses.

  • Spatial and temporal coverage

    Horizontal:Globe
    Vertical:Stratosphere
    Time:One year (1997)

  • Spatial and temporal resolution

    Latitude:5 degrees (from -87.5 to 87.5, negative latitudes in Southern Hemisphere)
    Longitude:7.5 degrees (from 0 to 352.5 east)
    Height:The model uses 18 isentropic levels. The vertical coordinate in the data files is the globally averaged altitude - to make it easier to generate plots with NetCDF software. The real lat/lon-dependent altitude is given in the ALT field. The THETA field gives the real model theta levels (which are constant with latitude/longitude).
    Time:1 month (one model output near the middle of each month)

  • Data file format

    NetCDF (binary)

  • Data volume, number of files

    12 files, 5.35 Mbytes each

  • File names

    slim_ddmmyy_198.ncwhere yy = 97
    mm = 01, 02, ..., 11, 12
    dd = day number near middle of month (10, 11, 12, 13, 18 or 19)

  • File contents

    The 3-D fields listed under Recorded variables, as computed at one output time.

    Example: slim_100897_198.nc contains the model output on the 10th of August 1997.
