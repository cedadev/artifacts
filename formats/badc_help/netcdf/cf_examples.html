BADC: CF Conventions - examples of global attributes

CF conventions are guidelines and recommendations as to where to put information, and they provide advice as to what information you might want to include. Extra attributes are not outlawed and the BADC recommends you include as much information as possible.

CF conventions make one global attribute mandatory:

Conventions "CF-1.0"

For the source, if the data are model generated, source should name the model and the version number. If the data are observational, source should characterize them, e.g. surface observation, radiosonde, satellite.

With the history attribute, well-behaved generic software will automatically append its name, input parameters, and a timestamp.

Here we give three examples of datasets using the CF conventions, with the recommended global attributes: instrumental data, gridded model data, and surface data.
